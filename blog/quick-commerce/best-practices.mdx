  src="https://assets.autorender.io/LOKVTtKVGb/w_auto/doc1/product.jpg"
  srcset="

## Leverage Edge Caching

### Do:

- Use width buckets (320, 480, 720, 1080, 1440, 1920)
- Stick to preset recipes
- Use consistent transformation parameters

### Don't:

- Use random or frequently changing dimensions
- Generate unique URLs for each user
- Over-parameterize transformations

## Mobile Optimization

Quick commerce is mobile-first. Optimize accordingly:

```bash
# Mobile-optimized: smaller size, WebP format, optimized quality
https://assets.autorender.io/LOKVTtKVGb/w_480,h_480,c_fill,q_80,f_webp/doc1/product.jpg
```

**Key mobile optimizations:**
- Smaller dimensions (480px max for thumbnails)
- WebP format for 25-30% size reduction
- Quality setting 80-85 (sweet spot for size/quality)
- Lazy loading for below-the-fold images

## Width Bucketing Strategy

AutoRender automatically rounds image widths to predefined sizes (320, 480, 720, 1080, 1440, 1920px). This provides:

**Better Cache Hit Rates**: When multiple requests ask for similar sizes (e.g., 650px and 680px), AutoRender serves the same cached 720px variant.

**Faster Delivery**: Pre-optimized images at standard sizes are served instantly from edge caches.

**Cost Reduction**: Fewer unique transformations mean lower processing costs.

```bash
# User requests w_650, AutoRender serves w_720 (nearest bucket)
https://assets.autorender.io/LOKVTtKVGb/w_650/doc1/product.jpg
```

## Device-Aware Default Widths

AutoRender automatically detects device types and applies appropriate default widths:

- **Mobile devices**: 480px default (perfect for product cards)
- **Tablets**: 720px default (ideal for detail views)
- **Desktop**: 1080px default (full-quality experience)
- **High-DPI displays**: Automatically scaled for retina quality

This means you don't need to maintain separate image URLs for different devices—AutoRender handles it automatically.

## Transformation Recipes

Quick commerce platforms often need consistent image transformations across thousands of products. AutoRender's recipe system lets you define common transformations once and reuse them everywhere:

```bash
# Define once: t_product-thumbnail → c_fill,w_300,h_300,p_c
# Use everywhere:
https://assets.autorender.io/LOKVTtKVGb/t_product-thumbnail/doc1/product.jpg
```

This ensures consistency, improves cache hit rates, and simplifies your codebase.
