## Overview

AutoRender Recipes (preset tokens) allow you to store and reuse common transformation combinations using simple `t_` tokens. Instead of writing long transformation strings repeatedly, you can use a short recipe name that automatically expands to the full transformation.

### URL Structure

AutoRender URLs with recipes follow this structure:

```bash
https://assets.autorender.io/LOKVTtKVGb/t_thumbnail-2/docs/example/porsche.jpg
```

## How Recipes Work

Recipes use the `t_` prefix followed by a preset name. When AutoRender encounters a recipe token, it automatically expands it to the full transformation string stored in your workspace configuration.

### Recipe Expansion Process

1. **Token Detection**: AutoRender detects `t_` prefix in the transformation string
2. **Lookup**: Searches for the recipe in your workspace's key-value store
3. **Expansion**: Replaces the token with the stored transformation string
4. **Processing**: Applies the expanded transformations along with any additional parameters

## Benefits of Using Recipes

### 1. Simplified URLs

Instead of long transformation strings, use short, memorable recipe names:

Without recipe (long URL):

```bash
https://assets.autorender.io/LOKVTtKVGb/w_400,h_400,c_crop,ar_1:1,p_face,e_auto_enhance,e_warmth:25,e_shadows:30,e_structure:25/docs/example/porsche.jpg
```

With recipe (short URL):

```bash
https://assets.autorender.io/LOKVTtKVGb/t_profile-picture/docs/example/porsche.jpg
```

### 2. Consistency

Ensure consistent transformations across your application:

All product thumbnails use the same recipe:

```bash
https://assets.autorender.io/LOKVTtKVGb/t_product-thumbnail/product-1.jpg
https://assets.autorender.io/LOKVTtKVGb/t_product-thumbnail/product-2.jpg
https://assets.autorender.io/LOKVTtKVGb/t_product-thumbnail/product-3.jpg
```

### 3. Easy Updates

Update transformations in one place - all URLs using the recipe automatically use the new settings:

Update the recipe definition once, and all URLs using `t_product-thumbnail` automatically get the update.

### 4. Reduced Errors

Fewer typos and mistakes when using short recipe names instead of long transformation strings.

## Creating and Managing Recipes

Recipes are stored per workspace in AutoRender's key-value store. The format is:

```
{workspace_id}/t_{recipe_name} = {transformation_string}
```

### Recipe Naming

- Use descriptive names: `t_product-thumbnail`, `t_hero-banner`, `t_profile-picture`
- Use hyphens for multi-word names: `t_product-detail`, `t_category-banner`
- Keep names concise but clear

### Recipe Content

Recipes can contain any valid AutoRender transformations:

- Basic transformations: `w_800,h_600,c_fill`
- Effects: `e_saturation:30,e_contrast:15`
- Multiple parameters: `w_400,h_400,c_crop,ar_1:1,p_face`
- Complex combinations: `e_grayscale,e_brightness:-93,e_extend:10:10:10:10:black`

<Info>
  Contact AutoRender support to create and manage custom recipes for your workspace. Recipes are workspace-specific and can be updated at any time.
</Info>

## Best Practices

### Recipe Naming

- Use descriptive, consistent names
- Follow a naming convention (e.g., `t_product-{size}`, `t_category-{type}`)
- Keep names short but clear
- Use hyphens, not underscores

### Recipe Organization

- Group related recipes (e.g., all product-related recipes)
- Document recipe purposes
- Version recipes if needed (e.g., `t_product-thumbnail-v2`)

### Performance

- Recipes are cached for fast lookup
- Combine recipes with additional parameters when needed
- Use recipes consistently across your application

### Maintenance

- Review recipes periodically
- Update recipes when design requirements change
- Test recipe changes before deploying
- Document recipe definitions for your team
