## What to include in docs for SDK users

For each framework, keep docs focused on these four blocks:

1. **Install**: exact package + version strategy
2. **Initialize**: workspace + auth/public key setup
3. **View SDK usage**: render transformed image/video URLs in components
4. **Upload SDK usage**: client-side upload flow + secure token guidance

<Note>
  Avoid copy-heavy boilerplate. Keep one minimal example per framework and link to a live sandbox.
</Note>

## Suggested doc structure per framework

| Section | Keep it concise |
| --- | --- |
| Install | `npm i ...` only |
| Configure | required env keys only |
| View example | one transformed media component |
| Upload example | one upload action + response handling |
| Production notes | auth boundary + limits + retries |

## Security note for upload SDK

Client apps should upload with short-lived signed tokens or scoped keys from your backend. Do not expose long-lived admin credentials in frontend code.
