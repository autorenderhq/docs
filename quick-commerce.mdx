## Overview

Quick commerce (Q-commerce) platforms require lightning-fast image delivery to provide exceptional user experiences. AutoRender's CDN-powered image optimization is designed to meet these demanding requirements, ensuring your product images load instantly across all devices and network conditions.

## Why AutoRender for Quick Commerce?

Quick commerce platforms face unique challenges:

- **Speed is critical**: Users expect instant page loads
- **Mobile-first**: Most orders come from mobile devices
- **High traffic spikes**: Flash sales and promotions create sudden traffic surges
- **Global reach**: Need consistent performance across regions
- **Bandwidth optimization**: Reduce data usage for cost savings

AutoRender addresses all these challenges with automatic optimization, edge caching, and intelligent image delivery.

## Key Features for Quick Commerce

### Automatic Width Bucketing

AutoRender automatically rounds image widths to predefined sizes (320, 480, 720, 1080, 1440, 1920px) for optimal caching. This means:

- **Better cache hit rates**: More requests match cached variants
- **Faster delivery**: Pre-optimized images serve instantly
- **Reduced processing**: Less on-the-fly transformation needed

```bash
# Original request: w_650
# AutoRender serves: w_720 (nearest bucket)
https://assets.autorender.io/LOKVTtKVGb/w_650/product-123.jpg
```

<img src="https://assets.autorender.io/LOKVTtKVGb/w_650/product-123.jpg" alt="Width bucketing example" class="ar-url-image" />

### Device-Aware Default Widths

AutoRender automatically detects device types and applies appropriate default widths:

- **Mobile devices**: 480px default
- **Tablets**: 720px default
- **Desktop**: 1080px default
- **High-DPI displays**: Automatically scaled

This ensures users always get appropriately sized images without manual configuration.

## Best Practices for Quick Commerce

### 1. Optimize Product Images

**Use appropriate dimensions:**
- **Thumbnails**: 200x200px or 300x300px
- **Product cards**: 400-600px width
- **Product detail**: 800-1200px width
- **Hero banners**: 1920px width

**Example - Product thumbnail:**
```bash
https://assets.autorender.io/LOKVTtKVGb/w_300,h_300,c_fill,p_c/product-123.jpg
```

<img src="https://assets.autorender.io/LOKVTtKVGb/w_300,h_300,c_fill,p_c/product-123.jpg" alt="Product thumbnail" class="ar-url-image" />

**Example - Product detail:**
```bash
https://assets.autorender.io/LOKVTtKVGb/w_1200,c_fit/product-123.jpg
```

<img src="https://assets.autorender.io/LOKVTtKVGb/w_1200,c_fit/product-123.jpg" alt="Product detail" class="ar-url-image" />

### 2. Implement Responsive Images

Use AutoRender's automatic width detection with responsive image tags:

```html
<img 
  src="https://assets.autorender.io/LOKVTtKVGb/w_auto/product-123.jpg"
  srcset="
    https://assets.autorender.io/LOKVTtKVGb/w_480/product-123.jpg 480w,
    https://assets.autorender.io/LOKVTtKVGb/w_720/product-123.jpg 720w,
    https://assets.autorender.io/LOKVTtKVGb/w_1080/product-123.jpg 1080w
  "
  sizes="(max-width: 480px) 100vw, (max-width: 720px) 50vw, 33vw"
  alt="Product name"
/>
```

### 3. Leverage Edge Caching

**Use consistent transformation parameters:**
- Stick to width buckets (320, 480, 720, 1080, 1440, 1920)
- Use preset recipes for common transformations
- Avoid random or frequently changing parameters

**Cache-friendly URL:**
```bash
https://assets.autorender.io/LOKVTtKVGb/w_720,h_720,c_fill/product-123.jpg
```

**Cache-unfriendly URL (avoid):**

### 4. Optimize for Mobile

**Prioritize mobile performance:**
- Use smaller dimensions for mobile views
- Implement lazy loading
- Use WebP format for better compression
- Consider lower quality settings for thumbnails

Preload critical product images for faster perceived performance:

### 7. Use Recipes for Consistency

Create and use recipes for common product image sizes:

```bash
# Thumbnail recipe
t_thumbnail-2 → c_fill,w_200,h_200,p_c

# Product card recipe  
t_product-card → c_fill,w_400,h_400,p_c

# Detail page recipe
t_product-detail → w_1200,c_fit
```

## Performance Optimization Tips

### Reduce Image Sizes

- Use appropriate quality settings (q_80-90 for most cases)
- Enable WebP format for modern browsers
- Crop images to exact dimensions needed
- Remove unnecessary metadata

### Minimize Transformations

- Use preset recipes instead of long transformation strings
- Cache transformed images on your CDN
- Pre-generate common image variants

### Monitor Performance

- Track image load times
- Monitor cache hit rates
- Measure bandwidth usage
- Analyze user experience metrics
